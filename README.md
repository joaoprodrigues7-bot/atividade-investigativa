# Quadro Comparativo de Licenças

| **Licença**   | **Finalidade Principal**                              | **Permissões**                                      | **Restrições**                                              | **Projetos Conhecidos**                |
|---------------|--------------------------------------------------------|----------------------------------------------------|-------------------------------------------------------------|----------------------------------------|
| **MIT**       | Permitir ampla liberdade de uso com poucas restrições | Uso, modificação, distribuição, uso comercial      | Deve manter o aviso de copyright e a licença                 | jQuery, Node.js                        |
| **GPL v3**    | Garantir liberdade total do código                    | Uso, modificação, distribuição, uso comercial      | Código derivado também deve ser GPL (copyleft)               | GIMP, Bash                             |
| **Apache 2.0**| Proteção de patente e uso empresarial                 | Uso, modificação, distribuição, uso comercial      | Manter aviso de licença e mudanças                           | Android, Apache Spark                  |
| **BSD 3-Clause** | Reutilização com poucas restrições                   | Uso, modificação, distribuição                     | Proíbe uso do nome dos autores sem permissão                  | FreeBSD                                |
| **MPL 2.0**   | Mistura de código aberto e fechado                    | Uso, modificação, distribuição                     | Trechos modificados devem ser abertos                        | Firefox                                |

Perguntas Reflexivas
1. Qual licença você adotaria se fosse criar um projeto aberto em equipe?
Escolheria a licença MIT pela sua simplicidade e permissividade, incentivando a colaboração e o uso comercial sem barreiras.

2. Como a escolha da licença impacta o trabalho colaborativo?
Licenças abertas incentivam a contribuição e uso amplo, enquanto licenças com copyleft (como GPL) garantem que o código sempre permaneça livre. A escolha define o tipo de comunidade que será formada.
